# IMPL-HO — 세션 시작 리추얼 원샷화(WP2) + handoff-lint 가드(WP3)

## 1. Objective

새 세션의 3회 도구 왕복(`bun run status`·HANDOFF.md 읽기·`tasks/lessons.md` 인덱스 읽기)을
`SessionStart` hook 자동 주입으로 대체한다(WP2). 아울러 HANDOFF.md 비대화를 조기 경고하는
`handoff:lint` 가드를 추가한다(WP3). 스펙은 아래 확정본이며 재설계 금지.

핵심 규약(혼동 금지):
- **아키텍트 세션 bootstrap 주입** = SessionStart hook이 stdout으로 JSON을 뱉어
  `additionalContext`를 주입 → Claude Code 컨텍스트에 삽입된다.
- 이는 **브릿지 워커 hook-sensor(stdout 비움·소켓 송신) 규약과 정반대**다. 두 경로를 섞지 말 것.

## 2. Files & 변경

### 2.1 `scripts/session-status.ts` (리팩터 + 신규 함수)
1. 현 CLI 출력을 **그대로 유지**하면서 상태표 생성부를 `export function buildStatus(): string`로
   추출한다. 현재 모듈 최상위에서 즉시 `console.log(...)`하는 구조를 `main` 가드로 바꾼다 —
   즉 `if (import.meta.main) { console.log(buildStatus()); … }`. `buildStatus()`는 지금
   `console.log`에 넘기던 문자열 전체(표 + `Files:`/`Run:` 꼬리줄까지)를 리턴한다.
2. `export function checkHandoffBudget(): string | null` 신설:
   - HANDOFF.md **상단 80줄의 바이트 수 > 8192**, 또는 **파일 전체 바이트 ≥ 16384**면 경고
     문자열을 리턴한다. 경고 문자열:
     `⚠ HANDOFF top-80 = NB > 8192 — 종결 웨이브를 docs/HANDOFF_ARCHIVE.md로 이관(규칙 L15)`
     (여기서 `NB`는 실제 측정 바이트 수로 치환 — 예: `⚠ HANDOFF top-80 = 9001B > 8192 — …`.
      전체 위반이면 top-80 대신 전체 바이트를 명시하는 대응 문구로.)
   - 정상이면 `null`.
   - 바이트 측정은 UTF-8 바이트 기준(`Buffer.byteLength`/`Bun`의 바이트 길이). 문자수 아님.
3. CLI 출력 말미: `import.meta.main` 경로에서 `checkHandoffBudget()`가 non-null이면
   경고를 **1줄 추가** 출력한다(기존 표 뒤에). null이면 아무것도 추가하지 않는다.
4. `--lint` 인자: `process.argv`에 `--lint`가 있으면 **경고만** 출력하고
   위반 시 `process.exit(1)`, 정상이면 표를 출력하지 않고 `process.exit(0)`.
   (즉 `--lint`는 표를 찍지 않는 가드 모드. 위반 시 경고 1줄 + exit 1.)

### 2.2 `scripts/session-context.ts` (신설)
`--part state` | `--part lessons` 인자를 받아 `SessionStart` hook용 JSON을 stdout으로 낸다.

출력 형식(두 part 공통):
```json
{"hookSpecificOutput":{"hookEventName":"SessionStart","additionalContext":"<내용>"}}
```
- **`additionalContext`는 9,500자 하드캡.** 초과 시 말미를 절단하고 마지막에
  `…[truncated N chars — 원문 파일 참조]` 줄을 붙인다(N = 절단된 문자 수).
- 내용은 **사실 서술형 데이터**로 작성 — 명령형 지시문 금지(Claude Code가 명령형 주입을
  프롬프트-인젝션 방어로 무시할 수 있음). "다음을 하라" 류 문장 대신 상태·사실을 나열.

`--part state` 내용:
1. 센티널 1줄: `[LOOM-SESSION-CONTEXT v1 · state]`
2. `buildStatus()`(2.1에서 export) 상태표 원문
3. HANDOFF.md에서 **헤딩 앵커로 추출**한 두 섹션 원문:
   - `## One-line resume` 섹션 (헤딩 다음부터 다음 `## ` 헤딩 직전까지)
   - `## ⭐ Current action` 섹션 (동일 방식)
   (앵커가 없으면 해당 섹션은 생략 — 예외로 세션을 막지 않는다.)
4. `checkHandoffBudget()` 경고(non-null이면 1줄 추가).

`--part lessons` 내용:
1. 센티널 1줄: `[LOOM-SESSION-CONTEXT v1 · lessons]`
2. `tasks/lessons.md` **전문**.

**fail-open**: 함수 전체를 try/catch로 감싸고, 어떤 오류든(파일 없음·파싱 실패 등)
catch에서 **stdout 출력 없이 `process.exit(0)`**. 세션 시작을 절대 막지 않는다.
`session-status.ts`의 export를 import해 재사용한다(buildStatus/checkHandoffBudget).

### 2.3 `.claude/settings.json`
- 기존 `hooks.PreToolUse`(check-agent-model.ts) 블록 **보존**.
- `hooks.SessionStart` 추가:
  - matcher: `"startup|clear"` (compact·resume는 **의도적 제외** — resume은 후속 스파이크 재결정).
  - hooks 배열에 command 2개, 각 `"type": "command"`, `"timeout": 30`:
    - `bun run "$CLAUDE_PROJECT_DIR/scripts/session-context.ts" --part state`
    - `bun run "$CLAUDE_PROJECT_DIR/scripts/session-context.ts" --part lessons`
- 유효 JSON 유지(주석 금지).

### 2.4 `AGENTS.md`
- 세션 시작 리추얼 v2 표 **앞에** 분기 1문단 추가:
  > 컨텍스트에 `[LOOM-SESSION-CONTEXT` 센티널이 보이면 아래 표의 ①status·②HANDOFF·
  > ③lessons 인덱스 읽기는 **주입으로 대체됨 — 생략**한다(카테고리 lessons 로드·WORKFLOW
  > grep은 종전대로). 센티널이 없으면(훅 실패·Codex CLI 등 무훅 환경) 아래 v2 표 전체가 정본.
- 이어 1줄 명시:
  > 아키텍트 세션 bootstrap 주입(SessionStart stdout 주입)과 브릿지 워커 hook-sensor
  > (stdout 비움·소켓 송신)는 **반대 규약** — 혼동 금지.
- **v2 표 자체는 삭제 금지**(무훅 환경 정본으로 유지).

### 2.5 `package.json`
- `scripts`에 추가: `"handoff:lint": "bun run scripts/session-status.ts --lint"`.

## 3. Constraints
- `packages/*` 제품 코드 **절대 불변**.
- **커밋/푸시 금지** — 검증까지만. ship은 아키텍트 결정.
- `bun run status` **출력 계약 불변**(기존 표 그대로. 경고줄은 위반 시에만 말미 추가).
- biome lint 통과 코드 스타일.
- `session-context.ts`는 다른 스크립트에 side-effect import되어도 안전하도록
  실행 로직을 `import.meta.main` 가드 뒤에 둔다.

## 4. Verification (워커가 실행·결과 보고)
```
bun run status                                     # 기존 표 + (해당시) 경고줄
bun run handoff:lint; echo "exit=$?"               # 현행 HANDOFF는 예산 내면 exit=0
bun run scripts/session-context.ts --part state | bun -e 'const j=JSON.parse(await Bun.stdin.text()); const c=j.hookSpecificOutput.additionalContext; console.log("state chars:",c.length, c.length<=9500)'
bun run scripts/session-context.ts --part lessons | bun -e 'const j=JSON.parse(await Bun.stdin.text()); const c=j.hookSpecificOutput.additionalContext; console.log("lessons chars:",c.length, c.length<=9500)'
bun -e 'JSON.parse(await Bun.file(".claude/settings.json").text()); console.log("settings json ok")'
biome lint scripts/session-status.ts scripts/session-context.ts
```
최종 줄 마커(정확히 이 형식):
`[IMPL-HO-DONE] state=<chars> lessons=<chars> lint=<exit>`
