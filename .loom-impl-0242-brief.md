# [IMPL-0242] PLAN v0.24.2 구현 브리프 — Windows 실배포 결함 2건 수정 (grok pane)

당신은 Loom dogfood 룸의 **grok-impl 구현자**다. PLAN §0.24.2(approved R39)를 구현한다.
**시작 전 `docs/PLAN.md`의 `#### 0.24.2` 블록을 정독하라 — 스펙 정본이다.**

## Objective

Windows 실배포 결함 2건 수정(라이브 실증: durable 기동 후 첫 room create에서 relay 사망 2회 재현):
- **D1** persist 경로 가드의 POSIX 구분자 하드코딩 오탐 교정 — Windows에서 모든 스냅샷 쓰기가 거부되던
  버그(`persist.ts:389`가 `"/"` 하드코딩, `roomStatePath`의 `join()` 산출 백슬래시 경로와 불일치).
- **D2** relay 메시지 핸들러 uncaught 예외가 프로세스 전체를 죽이던 것을 try/catch로 차단(서버 생존만
  추가, 클라이언트 단위 fail-closed·create 롤백/rethrow는 무변경).
- **D9** VERSION 0.24.1 → **0.24.2**.

## Files

- `packages/relay/src/persist.ts`
  - `:23` import에 `sep` 추가: `import { join, dirname, basename, resolve, sep } from "node:path"`
  - `:389` 가드 교정: `path.startsWith(realState + "/")` → `path.startsWith(realState + sep)`
    (**의미론 불변 — 가드 완화 아님. POSIX에서 `sep === "/"`라 판정 동일, Windows에서만 `\` 정합**)

    로그 후 **계속**(throw 재전파 금지, 연결 유지 — 프로세스 생존).
- `packages/cli/src/index.ts:144` — `const VERSION = "0.24.2"`
- `packages/mcp-server/src/stdio.ts:381` — `serverInfo: { name: "loom", version: "0.24.2" }`
- `packages/relay/src/persist.test.ts`(또는 신규 테스트 파일) — 아래 테스트 2종

## 테스트 (R39 M-1 lock — 위반 금지)

**① containment 어서션** (D1):
- 적대적 `room.id` 군을 투입: 상대 이탈 `../../etc/x`, POSIX 절대경로 `/etc/passwd`, Windows 절대경로
  `C:\\Windows\\x`, 백슬래시 혼입 등.
- 각 케이스가 `saveRoomSnapshot` **쓰기 성공**하고, 산출 경로(`roomStatePath(realState, id)`)가
  **state dir 직하 16-hex `.json` 파일**임을 어서션(`path.startsWith(realState + sep)` 프리픽스 만족·
  파일명이 sha256 16-hex).
- **"거부됨(throws)" 어서션 금지.** `roomStatePath`(`persist.ts:67-69`)가 `room.id`를 sha256 16-hex로
  해시해 파일명을 만들므로 어떤 입력으로도 가드 거부 분기(`throw`)가 공개 표면에 도달 불가 —
  구현 불가 테스트를 시도하거나 가드/파생 코드를 변조하지 말 것(R39 M-1 자기모순 지적의 핵심).

**② D2 핸들러 크래시 가드**:
- `handleMessage`가 throw하는 상황을 투입(예: persist 실패를 주입한 create 메시지) → **서버 생존 +
  후속 요청 정상 처리** 확인(`server.integration.test.ts` 스타일).

## Constraints

- create의 **롤백·rethrow**(`room.ts:743-752`) **무변경** — D2는 프로세스 생존만 추가, create 실패
  의미론(byId/byCode delete·persist hook null·rethrow)은 그대로.
- 가드 술어 추출·`relative()` 재작성 등 **리팩터 금지**(D7) — `sep` 1글자 교정만.
- 와이어 프로토콜·스냅샷 파일 형식/버전 **무변경**.
- **git commit / push 금지** — 검증까지만, ship은 아키텍트 결정.
