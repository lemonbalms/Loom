# [IMPL-0250] PLAN v0.25.0 구현 브리프 — conv artifact fetch 자동 실행: 신규 MCP 툴 `conv_fetch` (scp transport v1)

당신은 Loom dogfood 룸의 **grok-impl 구현자**다. PLAN §0.25.0(approved R40)을 구현한다.
**시작 전 `docs/PLAN.md`의 `#### 0.25.0` 블록(53~110행)을 정독하라 — 스펙 정본이다.** 이 브리프는
정본의 실행 지침이며, 경로·시그니처는 실측으로 확정돼 있으나 **구현 전 인접 코드를 직접 정독**해
기존 스타일(에러 반환 셰이프·`ok`/`error` 유니언·`Bun.spawn` 패턴)을 그대로 따르라.

## 1. Objective

32k 초과 conv 산출물이 `artifacts[]` ref로 회신될 때, 지금까지 타워는 M-2 검증을 통과한 fetch **명령
문자열을 제시만** 하고 실행은 사람/에이전트에 맡겼다(0.23.1/R26 "제시까지"). 이 PLAN은 마지막 단계 —
**좌표(`convId`, `seq`, `index`)만으로 브릿지 노드에서 산출물 파일을 실제로 가져오는 신규 MCP 툴
`conv_fetch`** — 를 추가한다. 실행은 셸 미경유 argv 직접 spawn, 실행 직전 host·path 재검증(fail-closed),
수신 후 sha256 대조로 무결성 확정. **v1은 scp transport만** 자동화하고 git·자동 push는 제시-only 유예 존치.

**완료 정의:** `conv_fetch(convId, seq, index)` MCP 툴이 등록·배선되어, scp ref 좌표로 브릿지 노드에서
파일을 실제 회수하고, D4~D7 방어(재검증·containment·sha256 격리·가드)가 유닛 테스트로 고정되며,
6패키지 typecheck green + PLAN §0.25.0 테스트 계획 전수 통과. **커밋/푸시 금지 — 검증까지만.**

D1~D8 요약: **D1** 실행 표면 = `conv_fetch(convId, seq, index)` 단일(무게이트 자동 금지 — 방출·수신
자동 fetch 경로 신설 금지) · **D2** 호출자는 좌표만, 실행 파라미터(host/path/sha)는 서버측 저장분에서
해석 · **D3** scp transport만(git ref는 제시-only 유지) · **D4** 셸 미경유 argv + 실행 직전 재검증 ·
**D5** 목적지 `~/.loom/artifacts/<convId>/` 고정 + containment + 덮어쓰기 거부 · **D6** post-fetch sha256
대조·불일치 즉시 삭제 · **D7** 60s 타임아웃·10MiB 상한·동일 conv 직렬화 · **D8** VERSION 0.25.0 동기.

## 3. Interfaces

### `conv_fetch(convId, seq, index)` → 메타데이터-only 반환 (L-1)
반환에 **실행 argv(verbatim, M-25 echo 승계)·수신 경로·sha256 대조 결과·바이트 수**만 담고 **파일 내용은
포함하지 않는다**(대용량 산출물이 도구 반환에 재-인라인되어 32k/컨텍스트를 잠식하는 것 방지). 내용 인입은
호출자가 별도 read로 수행. 실패 시 `ok:false` + 사유(fail-closed 유니언 — 인접 `validateScpArtifactRef`
반환 셰이프 `{ok:true,...}|{ok:false,error}` 따르라).

### 실행 argv 조립 (D4)
`["scp", "-o", "BatchMode=yes", (scp가 "--" 지원 실측 시 "--"), "<host>:<path>", "<dest>"]`
- **셸 미경유** — `sh -c`·문자열 보간·POSIX 인용 없음. argv 배열 직접 `Bun.spawn`.
- `BatchMode=yes` — 비대화 강제(프롬프트 무한대기·자격 프롬프트 차단).
- **`--` 옵션 종료 실측(U2, L-3):** OpenSSH scp의 `--` 지원은 버전별 상이 가능. 구현 시
  `scp --` 동작을 실측하고(예: `scp -- 2>&1` 또는 `man scp` 확인), **지원/미지원 + 채택한 방어 형태를
  카드 회신에 명기**하라. 미지원 시 대체 방어 = host·path 선행 `-` 거부(이미 M-B/M-C가 커버) + 고정
  `-o` 옵션만으로 경계 유지.
- **`<host>:<path>`:** host = 실행-경로 재검증 통과분(M-B), path = `validateScpArtifactRef` 정규화 산출
  remote 경로(tilde-literal). 렌더 문자열(`presentScpFetchCommand`)을 파싱해 쓰지 말 것 — 별도 argv 조립.
- **`<dest>`:** D5 로컬 목적지(아래).

### 좌표→저장분 해석 (D2)
`convId`·`seq`·`index`로 서버측 저장분(conv-state M-D)에서 `artifacts[index]` ref를 읽는다. host·path·
sha256 등 실행 파라미터는 **호출자로부터 받지 않는다**. 미지 `convId`/미지 `seq`/범위 밖 `index`/
transport 불일치(git ref) → fail-closed 에러. 저장분 부재 → fail-closed.

## 4. Constraints (락 — 위반 금지, 전문 인용)

- **M-A (sha256 부재 ref 거부):** `ArtifactRefEntrySchema.sha256`은 optional(`conv-contract.ts:113-116`)
  이라 wire상 sha256 부재 ref가 합법이다. 무결성 대조 불가하므로 `conv_fetch`는 해당 ref를 **거부**하고
  그 ref는 **제시-only로 잔존**(자동 실행 대상 제외). **sha256 존재 = 자동 fetch의 전제 조건.**
- **M-B (host 실행-경로 `validateConvNodeHost` 전체 재적용):** host 매핑 저장분은 신뢰 불가 —
  `loadConvNodeHosts`(`conv-node-hosts.ts:86-102`)는 손편집 엔트리를 비엄격 로드(비공백이면 무검증 수용),
  `resolveConvNodeHost`(`:120-125`)와 `resolveScpHostForConv`(`conv-artifact-present.ts:25-29`)는 그 값을
  **재검증 없이 통과**시킨다. 따라서 실행 직전 `validateConvNodeHost`(`:54-66`)를 **전체 재적용**한다 —
  선행 `-` 거부 + charset allowlist(`CONV_NODE_HOST_RE`) + **`:` 배제**(host:path 단일 토큰 오파싱 방지)
  모두 포함. 위반 시 미실행. (단순 "선행 `-` 거부"는 이 상위 요건에 흡수됨.)
- **M-C (원격 path charset + `isSafeConvSuffix` 실행-경로 적용):** classic scp는 원격 path를 **원격
  셸이 해석**한다 — `validateScpArtifactRef`(`conv-contract.ts:212`)는 path의 `~/.loom/artifacts/<convId>/`
  **prefix만** 검사(`:230`)하고 접미 charset은 무제약이다. 따라서 spawn 직전 렌더 경로가 쓰는
  **charset allowlist(`CONV_SUFFIX_ALLOWED_CHARS` `[A-Za-z0-9._/-]`)와 `isSafeConvSuffix`(세그먼트 선행
  `-` 거부, `:271-274`)를 path의 convId-prefix 접미에 적용**한다(위반 시 미실행). **한계 명시:** argv 직접
  spawn은 **로컬 셸 보간만** 무력화한다 — 원격 path는 원격 sshd 셸이 해석하므로 이 charset 재검증이
  잔여 격차를 봉합한다(주석으로 명기).
- **M-D (fresh turn만 저장·replay 미갱신):** `isFreshPeerSeq`(`conv-state.ts:112`)를 통과한 신규 턴에서만
  ref를 기록하고, replay·순서-역전 턴은 **저장분을 갱신하지 않는다**(§2 파일 지침대로 조기 return 자리
  덕에 자연 충족). 위조·재전송 턴이 검증·회수된 좌표 ref를 덮어써 후속 `conv_fetch`를 오도하는 경로 차단.
- **D4 (셸 미경유 argv + 실행 직전 재검증):** ① `validateScpArtifactRef(convId, ref, resolveHost,
  artifactsRoot)` 재호출 — `artifactsRoot`는 `convArtifactsRootLiteral(convId)`(`conv-contract.ts:249`,
  tilde-literal, present 경로와 동일). ② host는 `resolveScpHostForConv(convId)`로 재해석 후 **M-B 재검증**.
  렌더 문자열 실행 경로 없음.
- **D5 (목적지 containment + 덮어쓰기 거부):** 수신 위치 = **로컬 expanded** `loomDir()/artifacts/<convId>/`
  하위 고정(remote 검증용 tilde-literal root와 **구분** — remote는 `~/.loom/...` 리터럴, 로컬 dest는
  `loomDir()` 확장). ref path의 **파일명 성분(basename)만** 취해 dest 구성. 최종 경로를 realpath 해석해
  conv 디렉터리 내부 확인(**root 쪽도 realpath** — macOS `/var→/private/var` 오판 방지). containment 탈출
  → fail-closed. **기존 파일 존재 시 덮어쓰기 거부(fail-closed).**
- **D6 (post-fetch sha256):** 수신 후 로컬 파일 sha256 계산해 ref.sha256과 대조, **불일치 시 즉시 삭제
  (격리) 후 실패 반환.** `chars`는 보조 대조. sha256은 fetch **후** 무결성 검증 전용(fetch 방어 아님).
- **D7 (가드):** ① 타임아웃 60s — 초과 시 프로세스 강제 종료 + 부분 파일 격리. ② 수신 10MiB 상한
  (`MAX_WORKER_ARTIFACT_BYTES` = `conv-artifact-pack.ts:43` `10 * 1024 * 1024`와 정합), 초과 시 중단·격리.
  ③ 동일 `convId` 동시 `conv_fetch` **in-memory 직렬화**(목적지 경합·부분 파일 인터리브 배제).
- **L-2 (normalize 플랫폼 의존):** `validateScpArtifactRef`의 `normalizePath`(`conv-contract.ts:8,229` =
  `node:path` `normalize`)는 플랫폼별 구분자 변환(Windows `/`→`\`)이 있다. 목적지 조립·containment 비교에
  **POSIX 구분자 가정을 두지 말 것** — 주석·테스트로 명시(0.24.2 결함 1 POSIX 하드코딩과 동계열).
- **git transport ref는 제시-only 유지(D3):** `transport:"git"` ref는 `conv_fetch` 대상 아님 — 기존
  `artifactCommands[]` 제시 경로 무변경. transport 불일치(git ref에 `conv_fetch`) → fail-closed.
- **`conv_await` 반환 계약·기존 present 경로 무변경:** `conv-ops.ts:530-532` `artifactCommands` 조립,
  `presentArtifactCommands`/`presentScpFetchCommand`/`validateScpArtifactRef`/`resolveConvNodeHost` 기존
  스위트 전부 무회귀. **wire·CONV_SPEC §5.3 스키마 무변경.**
- **커밋/푸시 금지 — 검증까지만, ship은 아키텍트/오너 결정.** `docs/plan_review.md` 편집 금지.

## 5. Verification (PLAN §0.25.0 테스트 계획 전수 — 99~108행)

유닛 테스트(`conv-artifact-fetch.test.ts` 및 필요 시 conv-state/conv-ops 테스트)로 아래 전수:
- **argv 조립 검증** — 셸 미경유 argv 배열(문자열 보간·인용 부재), 렌더 문자열 실행 경로 없음.
- **재검증 fail-closed** — conv→node 매핑 부재 → 미실행 + 사유(승계 락 ②) · containment 위반 path →
  fail-closed(root 쪽 realpath 포함) · **host 전체 재검증 실패(선행 `-`·내장 `:`·charset 위반) → 미실행
  (M-B)** · **path charset/`isSafeConvSuffix` 위반(convId-prefix 접미) → 미실행(M-C)**.
- **sha 불일치 격리** — 수신 파일 sha256 ≠ ref.sha256 → 즉시 삭제 + 실패(D6) · **sha256 부재 ref →
  거부(제시-only 잔존, M-A)**.
- **턴별 ref 저장(M-D)** — fresh turn 저장 후 `(convId, seq, index)` 조회 성공 · replay/순서-역전 턴이
  저장분 미갱신 → 위조 덮어쓰기 거부 · **containment/normalize 플랫폼 구분자 회귀(L-2)**.
- **덮어쓰기 거부** — 목적지 동일 파일 기존 존재 → fail-closed(D5).
- **좌표 검증** — 미지 `convId`/미지 `seq`/범위 밖 `index`/transport 불일치(git ref) → fail-closed(D2·D3).
- **git transport 제시-only 유지** — git ref는 `conv_fetch` 대상 아님, 기존 `artifactCommands[]` 무변경(락 ④).
- **가드** — 타임아웃 초과 종료+격리(D7①) · 10MiB 초과 중단+격리(D7②) · 동일 conv 동시 직렬화(D7③).
- **무회귀** — 기존 present/`validateScpArtifactRef`/`resolveConvNodeHost` 스위트 · `conv_await` 반환
  계약(`artifactCommands[]`) 무변경(D1).
