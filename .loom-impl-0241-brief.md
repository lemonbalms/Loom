# [IMPL-0241] PLAN v0.24.1 구현 브리프 — `loom relay` 포그라운드 durable 배선 (grok pane)

당신은 Loom dogfood 룸의 **grok-impl 구현자**다. PLAN §0.24.1(approved R38)을 구현한다.
**시작 전 `docs/PLAN.md`의 `#### 0.24.1` 블록을 정독하라 — 스펙 정본이다.**

## Objective

`loom relay` **포그라운드 분기**(`packages/cli/src/index.ts:3163-3233`)가 로컬 데몬과 동일하게
**durable RoomRegistry**로 기동하도록 배선을 이식한다. 현행 근인: 포그라운드 분기가 `RelayServer`를
`registry` 옵션 없이 생성 → `server.ts:59` 무인자 `RoomRegistry`(`stateDir=null·durable=false`,
`room.ts:674-677`)로 떠서 relay 재시작 시 룸·초대코드·멤버십 소실. 영속화·기동 복원·멤버십 복원
체인은 전부 기구현·정상 — **배선만 이식(PATCH)**.

## 작업 항목 (PLAN Decisions)

- **D1 (배선)**: 포그라운드 분기가 env(`LOOM_RELAY_EPHEMERAL`/`LOOM_RELAY_STATE_DIR`, 기본
  durable = `defaultRelayStateDir()`)로 결정한 `RoomRegistry`를 생성해 `RelayServer`에 `registry`로
  전달. 데몬 경로(`packages/relay/src/cli.ts:38-80`)와 동일 의미론.
- **D2 (셧다운 훅)**: 포그라운드 분기에 `SIGINT`/`SIGTERM` 핸들러로 `registry.close()` 호출 후 종료
  (데몬 경로 `cli.ts` shutdown 훅과 동형). 훅 부재 시 Ctrl-C가 stale 락 창(≈5s)을 남긴다 — durable화와
  동반 필수.
- **D3 (실패 시 fail-closed 유지)**: durable 배선 실패(stateDir 락 획득 불가 등) 시 데몬 경로와 동일하게
  `console.error` + `exit 1` — **자동 ephemeral 폴백 금지**. 추가로 escape-hatch 힌트 1줄 출력
  (예: "set LOOM_RELAY_EPHEMERAL=1 to run without persistence"). 손상 스냅샷은 방 단위 skip+백업+로그
  후 기동 계속(persist.ts 기존 거동) — 이는 D3 fail-closed 대상 아님.
- **D6 (공유 헬퍼)**: env→옵션 판정 로직을 relay 패키지 내 공유 헬퍼로 추출해 데몬 경로와 포그라운드
  분기가 공유. **아래 M-1 lock 준수 필수.**
- **D9 (VERSION 동기)**: `0.24.0` → **`0.24.1`** — CLI `packages/cli/src/index.ts:144`
  (`const VERSION = "0.24.0"`) + MCP serverInfo `packages/mcp-server/src/stdio.ts:381`
  (`serverInfo: { name: "loom", version: "0.24.0" }`) 동시 갱신(0.23.7 미갱신 사고 선례).
- **D11 (기동 상태 로그)**: 포그라운드 기동 시 `State: durable (<dir>)` 또는 `State: ephemeral …`
  출력(데몬 경로 `cli.ts:87-89`와 동형) — 조용한 ephemeral 기동 진단 가시성.

## Interfaces (R38 M-1 lock — 위반 금지)

공유 헬퍼(예: `resolveRegistryOptionsFromEnv()`)의 소관은 **env 판정 → `RoomRegistryOptions`
(`{ ephemeral: true }` 또는 `{ stateDir }`) 반환까지만**이다.

- `new RoomRegistry(...)` 생성 · `try/catch` · 에러 문구 · `process.exit(1)`은 **각 호출자(데몬 경로·
  포그라운드 분기)가 각자 유지**한다 — 헬퍼로 흡수 금지.
- `packages/relay/src/cli.ts`의 기존 **에러 문구·경로·관측 가능 동작은 무변경**이어야 한다(헬퍼 도입이
  데몬 경로의 try/catch·exit·State 로그를 재배치·재문구화하지 않는다).
- `RoomRegistryOptions` 타입은 `packages/relay/src/room.ts:652`에 기존재 — 신규 정의 금지, 재사용.

## Constraints

- `persist.ts`·`room.ts`·`server.ts` **로직 무변경**(배선만 — 신규 export 추가는 허용).
- 클라이언트 측(`packages/host/src/relay-bindings.ts` 등) **무변경**.
- 와이어 프로토콜·스냅샷 파일 형식/버전 **무변경**.
- **git commit / push 금지** — 검증까지만, ship은 아키텍트 결정.
