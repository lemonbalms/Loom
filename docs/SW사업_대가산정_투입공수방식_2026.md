## 2. 대가산정 방식 개요

SW사업 수명주기별로 산정 방식이 다르다.

| 단계 | 주요 산정 대상 | 주요 방식 |
|------|----------------|-----------|
| 기획 | ISP, ISP/BPR, EA/ITA, ISMP, 정보보안 컨설팅 | 컨설팅 업무량 방식 **또는** 투입공수 방식 |
| 구현 | 소프트웨어 개발비, AI 도입비, 재개발비 | **기능점수(FP) 원칙**, 예외 시 투입공수 |
| 운영 | 유지관리, 운영, 보안관제, 재개발 | 요율제, 투입공수, 고정·변동비, SLA 등 |

## 3. 투입공수방식 산정식 (핵심)

### 3.1 기본 공식

투입공수방식 적용 시(기획 컨설팅 일부, SW 운영·보안관제, 기능점수 적용 곤란한 개발 등):

```
투입공수방식 대가
  = 직접인건비
  + 제경비
  + 기술료
  + 직접경비
```

### 3.2 항목별 정의 및 요율

| 항목 | 정의 | 산식 / 요율 |
|------|------|-------------|
| **직접인건비** | 실제 투입 인력에 대한 인건비 | 직무별 투입공수(M/M) × IT직무별 SW기술자 평균임금 |
| **제경비** | SW 산업 특수성을 반영한 간접비 | 직접인건비 × **144% ~ 154%** |
| **기술료** | 기술 제공·관리 대가 (엔지니어링 대가기준 준용) | (직접인건비 + 제경비) × **20% ~ 40%** |
| **직접경비** | 해당 사업에 직접 소요되는 경비 | 실비·가격조사 등으로 별도 산정 |

가이드 원문 요지:

```
제경비 = 직접인건비의 144 ~ 154%
기술료 = (직접인건비 + 제경비)의 20 ~ 40%
```

### 3.3 직접인건비 계산

```
직접인건비 = Σ (직무별 투입공수 × 해당 직무 평균임금)
```

- 투입공수 단위: 통상 **M/M (Man-Month)**
- 단가: KOSA 공표 **IT직무별 SW기술자 평균임금**
- 과거 특급·고급·중급·초급 등급 단가가 아니라 **직무 기준**으로 산정

### 3.4 배수 감각 (직접경비 제외)

직접인건비를 100으로 둘 때:

| 조합 | 제경비 | 기술료 | 합계 | 직접인건비 대비 |
|------|-------:|-------:|-----:|----------------|
| 하한 (제경비 144% / 기술료 20%) | 144.0 | 48.8 | 292.8 | **약 2.93배** |
| 중간 (제경비 149% / 기술료 30%) | 149.0 | 74.7 | 323.7 | **약 3.24배** |
| 상한 (제경비 154% / 기술료 40%) | 154.0 | 101.6 | 355.6 | **약 3.56배** |

실무에서는 사업 성격·위험·기술 난이도를 고려하여 제경비율·기술료율을 **범위 안**에서 결정한다.

### 3.5 산정 절차 (요약)

| 단계 | 내용 | 산출물 |
|------|------|--------|
| 1. 사전준비 | 대상 업무 범위·요구사항 확정 | 업무 범위, 요구사항 |
| 2. 투입공수 산정 | 업무 특성에 따른 투입 직무·인원·기간·투입률 결정 | IT직무별 투입공수 |
| 3. 직접인건비 계산 | 직무별 공수 × 평균임금 | 직접인건비 |
| 4. 제경비·기술료 계산 | 요율 적용 | 제경비, 기술료 |
| 5. 직접경비 산정 | 사업 직접 경비 | 직접경비 |
| 6. 대가 합산 | 4개 항목 합산 | 총 대가 (부가세 별도/포함 구분) |

### 3.6 유의사항

- 투입공수로 산정한 공수는 **예산·예정가격 산정용**이다.
- 이를 사업 수행 중 **실제 투입인력 관리 기준으로 강제 적용하지 않는다**.
- 공수 산정은 경험·주관 의존도가 있으므로, 필요 시 관련 기업 견적으로 적정성을 보강한다.

## 4. 소프트웨어 개발비 산정

### 4.1 원칙: 기능점수(FP) 방식

소프트웨어 개발비는 **기능점수 방식이 원칙**이다.

```
SW개발비 = 개발원가 + 직접경비 + 이윤
```

| 항목 | 산식 | 비고 |
|------|------|------|
| 보정전 개발원가 | 기능점수 × 기능점수당 단가 | 단가 **605,784원/FP** |
| 보정후 개발원가 | 보정전 개발원가 × 보정계수 | 규모, 연계복잡성, 성능, 운영환경 호환성, 보안성 등 |
| 이윤 | 개발원가 × **25% 이내** | 상한 규제 |
| 직접경비 | 시스템 사용료, 개발도구 사용료 등 | 실비 |
| 제경비·기술료 | **별도 산정하지 않음** | FP 단가에 상응 항목 반영 |

#### 기능점수 방식 특징

- 사용자에게 제공되는 기능을 정량 측정 (ISO/IEC 14143 계열)
- 공급자 구현 방식이 아니라 **수요자 요구 기능** 관점
- 정통법(기능별 복잡도) / 간이법(평균 복잡도) 구분

### 4.2 예외: 투입공수 방식 (개발비)

기능점수 적용이 불합리한 경우에 한해, 발주자 판단으로 투입공수를 적용할 수 있다.

**적용 가능 유형 (가이드 예시)**

- 홈페이지 디자인, 웹 접근성 개선, 동영상 등 콘텐츠 관련 정보화사업
- R&D 성격의 소프트웨어 개발 사업
- 사용자 식별 기능 규모 대비 내부처리 복잡도가 현저히 높아 FP 적용이 불합리한 경우
- 데이터 튜닝·최적화, 테스트 등 기능점수 산정이 불가능한 경우
- 소프트웨어 개발 관련 예산이 **5천만원 미만**(부가세 제외)인 사업

**예외 시 산식**

```
투입공수 방식 SW개발비
  = (투입인력수 × 투입기간 × 기술자 직무별 단가)
  + 제경비
  + 기술료
  + 직접경비
```

- 제경비: 직접인건비의 **144% ~ 154%**
- 기술료: (직접인건비 + 제경비)의 **20% ~ 40%**
- 기능점수 방식의 「이윤 25%」와는 **별개 체계**

### 4.3 기능점수 vs 투입공수 비교 (개발비)

| 구분 | 기능점수(FP) 방식 | 투입공수 방식 |
|------|-------------------|---------------|
| 적용 | **원칙** | **예외** (발주자 판단) |
| 산정 관점 | 기능 규모 | 인력·기간 경험 추정 |
| 비용 구성 | 개발원가 + 직접경비 + 이윤 | 직접인건비 + 제경비 + 기술료 + 직접경비 |
| 간접·이윤 성격 | 이윤 ≤ 개발원가 25% | 제경비 144~154%, 기술료 20~40% |
| 제경비·기술료 | 단가에 포함 → 별도 없음 | 별도 계상 |

### 4.4 재개발비

```
재개발비 = 재개발원가 + 직접경비 + 이윤
이윤 = 재개발원가 × 25% 이내
```

(기능점수 기반 재개발 규모 산정 후 보정)

## 8. 계산 예시

### 8.1 투입공수 방식 (개발·운영 공통 구조)

**가정**

- 직무: 응용 SW 개발자  
- 투입: 3 M/M  
- 제경비율: 150%  
- 기술료율: 30%  
- 직접경비: 0원  

**계산**

| 단계 | 계산 | 금액 (원) |
|------|------|----------:|
| ① 직접인건비 | 3 × 7,754,124 | 23,262,372 |
| ② 제경비 | 23,262,372 × 150% | 34,893,558 |
| ③ 기술료 | (23,262,372 + 34,893,558) × 30% | 17,446,779 |
| ④ 직접경비 | — | 0 |
| **합계** | ①+②+③+④ | **75,602,709** |

*(부가세 별도)*

## 10. 실무 체크리스트

- [ ] 사업 단계(기획 / 구현 / 운영)와 산정 방식(업무량 / FP / 투입공수)을 먼저 확정했는가
- [ ] 개발비에서 투입공수를 쓰는 경우, 가이드상 **예외 유형**에 해당하는가
- [ ] 직접인건비 단가가 **2026년 IT직무별 평균임금**인가
- [ ] 제경비율이 **144~154%** 범위인가
- [ ] 기술료율이 **(직접인건비+제경비)의 20~40%** 범위인가
- [ ] FP 개발비인 경우 제경비·기술료를 **중복 계상하지 않았는가**
- [ ] FP 개발비 이윤이 **개발원가 25% 이내**인가
- [ ] 직접경비 근거(실비, 견적, 대가기준)가 있는가
- [ ] 부가세 별도/포함을 명시했는가
- [ ] 산정 공수를 실투입 관리 기준으로 오해하지 않았는가
