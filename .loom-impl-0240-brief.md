## 1. Objective

PLAN 0.24.0 **D1–D9**를 구현한다 — 단독(1인 다-relay) 모드 기능화:

- **D1** `LoomSession` additive 스키마: `relayName?` + `relays?: Record<string, RelayBinding>` (top-level은 계속 "활성 바인딩" 정본).
- **D2** `loom relay use <name>` — 활성 바인딩 명시 전환(스태시→승격).
- **D3** `loom relay list` — 바인딩 목록(전체 peerId, 토큰·시크릿 미노출).
- **D4** `--relay-name` 플래그(`room create`/`join`) + mirror-on-save 불변식 + 이종-relay 파괴 가드.
- **D5** `loom relay local start|stop|status` — 로컬 relay 데몬 라이프사이클(`relay-daemon.ts` 조각 재사용).
- **D6** 순수 함수 배치(`relay-bindings.ts` 신설) + 얇은 CLI 핸들러.
- **D9** VERSION 0.23.12 → **0.24.0** (CLI + MCP serverInfo 동시).

핵심 데이터 모델: `RelayBinding = { roomId, roomName, inviteCode, peerId, peerSecret?, relayUrl, relayToken?, updatedAt }` — 현행 top-level 룸-바인딩 7필드의 이름 붙은 스냅샷.

## 3. Interfaces (의미론 — PLAN D2·D3·D5 요지)

- **`loom relay use <name>`** (D2): (i) 현 top-level 바인딩이 **존재(roomId 설정)할 때만** 스태시·명명 요구 적용 — 현 바인딩을 `relays[relayName]`에 스태시하되 `relayName` 미설정 시 **fail-closed 에러**(안내: `--as <현재이름>`으로 먼저 명명). (ii) `relays[<name>]` 부재 → 에러 + **가용 이름 목록** 출력. **자동 생성·자동 failover 없음.** (iii) 성공 시 `relays[<name>]`→top-level 승격, `relayName=<name>`, 기록은 **`saveSession`(0600) 경로로만**. (iv) 대상 프로필은 현행 해석 우선순위(`sessionPath:248-261`), `--profile` 병용 가능. (v) 라이브 sticky host(`.host.json` pid 생존) 감지 시 **경고 출력 후 전환 수행**(전환은 새 프로세스부터 적용 — 구동 중 프로세스 재시작 필요를 stdout에 명시).
- **`loom relay use <name> --as <name2>` / `loom relay use --as <name>`(단독)** (D2-vi, M-2): §4 M-2 참조.
- **`loom relay list`** (D3): 바인딩별 이름·`relayUrl`·`roomName`·**전체 peerId**(절대 truncate 금지)·활성 마커 출력. **`relayToken`·`peerSecret`은 어떤 출력에도 미포함.** 출력 전 바인딩 `relayUrl` 토큰-스트립(L-4).
- **`loom relay local start|stop|status`** (D5): start=health 선확인 idempotent spawn / stop=pid→kill→사망 확인→pid 파일 정리 / status=`isRelayUp`+pid+endpoint 표시. **기존 sub-없는 `loom relay` 무변경.** loopback=open / 비-loopback=토큰 필수(H-5) 무변경.
- **`room create`/`room join --relay-name <name>`** (D4): 성공 시 `relayName` 설정 + top-level 갱신과 **동시에 `relays[relayName]` 미러**. §4 M-1 가드 적용.

## 4. Constraints — 락 요건 (R37, 구현이 반드시 지켜야 함)

> 아래는 PLAN D2·D4·D8 문안의 구현-필수 형태. 정본은 PLAN이며, 모호하면 PLAN 문안 자구를 따른다.

- **M-1 (이종-relay 파괴 가드 — 명명·무명 공통, fail-closed):** 현 top-level 바인딩이 존재(roomId 설정)하고 **현 `relayUrl`(정규형)과 다른 relay를 향한** `room create`/`join`은 **`relayName` 설정 여부와 무관하게 `--relay-name` 부재 시 에러로 정지**. 안내: 무명 프로필 = `loom relay use --as <이름>` 선행 / 명명 프로필 = `--relay-name <새이름>` 지정. `--relay-name <새이름>` 지정 시 통과(새 이름으로 `relayName` 전환·미러, 구 이름 바인딩 보존). **동일-relay 재조인·바인딩 없는 신규 프로필은 현행 그대로 통과.**
- **M-2 (`--as` 진짜 멱등):** `--as <name>`(단독형·`use <target> --as` 병용 공통)은 `relays[<name>]`이 **부재하거나 현 top-level과 전 필드 동등이면 기록**(재실행 no-op 무해), **기존-상이면 fail-closed 에러**(기존 엔트리의 relayUrl·roomName 요약 표시 + 다른 이름 또는 `--force` 안내). **무조건 덮어쓰기 금지.** 단독형=전환 없이 명명만, 병용=명명 후 전환 순서.
- **L-1 (동일/이종 판별):** 양측을 **`parseRelayUrl().wsUrl` 정규형으로 정규화 후 문자열 동등**. 저장측 relayUrl은 `normalizeSession`이 이미 정규형 유지. 호스트 별칭(localhost vs 127.0.0.1)은 **과잉-이종 판정 = fail-closed 안전 방향으로 수용**(별칭 정규화 금지).
- **L-2 (빈 프로필 한정):** D2 스태시·명명 요구는 **현 top-level 바인딩 존재(roomId 설정) 시에만**. 빈 프로필은 미지-이름 에러(D2-ii)가 자연 커버.
- **L-3 (미러 소스 = 디스크 원본):** 스태시/미러의 소스는 **env 토큰 병합 전 디스크 원본** — `envRelayToken()` 병합본 스냅샷 금지(전환 왕복 시 교차-relay 토큰 오염 차단).
- **L-4 (미러 순서·토큰-스트립):** 미러는 `saveSession` 내 `normalizeSession` **이후** 실행(맵에 H-6-clean URL만 기록). `list` 출력 전 바인딩 `relayUrl` 토큰-스트립(맵 엔트리는 `normalizeSession` 비통과라 손편집 `?token=` 임베드 노출 갭 차단).
- **L-5 (왕복 보존 테스트):** `relays` 포함 세션의 **load→save 왕복 보존** 케이스 추가(구-스키마 무변형 통과 회귀와 별개).
- **PLAN 테스트 행(D6/테스트 항목)의 케이스를 전부 구현하라** — §5에 열거.

## 5. 테스트 케이스 (PLAN 테스트 행 — 전부 구현)

- **D2** `relayName` 미설정 시 `use` fail-closed 에러 + `--as` 안내 · 미지 `<name>` → 에러 + 가용 목록 · 정상 전환 시 스태시→승격 왕복·`relayName` 갱신·저장이 0600 경로만 경유 · sticky host 감지 시 경고 후 전환.
- **D3** list 출력에 **전체 peerId** 포함 · **`relayToken`·`peerSecret` 미노출**(D8-a).
- **M-2** `--as` 동일-내용 재실행 → no-op(진짜 멱등) · **기존-상이 엔트리 충돌 → fail-closed 에러**(`--force` 없이 덮어쓰기 불가).
- **D4** `--relay-name` create/join 시 top-level+`relays[name]` 동시 갱신·정합 · mirror-on-save 불변식(relayName 설정 후 saveSession 경유 임의 top-level 갱신이 맵에 즉시 반영).
- **M-1** 명명 프로필 × 이종-relay × 무-플래그 create/join → **fail-closed 에러** · × `--relay-name <새이름>` → 새 이름 기록 + 구 이름 바인딩 보존 · 무명 바인딩 × 이종-relay → fail-closed 에러 + `--as` 안내 · **동일-relay 재조인(명명·무명) → 현행 통과(하위호환 회귀)**.
- **L-3** 미러 소스 = env 토큰 병합 전 디스크 원본(환경 토큰이 바인딩에 미영속).
- **L-5** `relays` 포함 세션 load→save 왕복 보존.
- **D5** local start idempotent(중복 health-up no-op) · stop pid 사망 확인 · status 표시.
- **회귀** 기존 top-level 소비자(`relayClientOptsFromSession`) 무변경.
