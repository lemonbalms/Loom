## 1. 필터 코드 현황

전부 `packages/host/src/bridge-runtime.ts` 한 파일. 함수 2개가 핵심:

### `stripTuiChrome(text)` — `:200-227` (export)
conv inline text + 카드 summary **입력**에만 적용. 카드 `output` 본문에는 절대 미적용(R31 M-1, §5 참조).
줄 단위로 순회하며 다음 chrome 가족을 drop(빈 줄·말미 공백 정리 포함):

| 가족 | 코드 위치 | 조건 |
|------|-----------|------|
| box-drawing 전용 줄 | `:170` `BOX_DRAWING_LINE_RE` / 판정 `:209` | 줄 전체가 박스/보더 문자 |
| 콘텐츠-포함 box 상태줄 | `:190` `BOX_STATUS_LINE_RE = /^╰─.*─╯$/u` / `:210` | grok `╰─ Grok 4.5 (high) ─╯` — 전체-매치만 |
| composer 프롬프트 | `:173` `COMPOSER_PROMPT_RE = /│\s*❯/` / `:211` | Claude/Ink `│ ❯ …` |
| bare composer `❯` | `:214` | `trimmed === "❯"` 전체-매치만 (`❯ 텍스트` 보존) |
| key-hint substring | `:176-184` `KEY_HINT_MARKERS` / `:215` | `Shift+Tab:mode`·`Ctrl+.:shortcuts`·`Ctrl+c:cancel`·`⏵⏵ auto mode on`·` · /effort` |
| **claude 상태줄** | **`:218` (인라인 조건, 정규식 상수 없음)** | **`line.includes(" │ ") && (line.includes("⚡") \|\| line.includes("🧠"))`** |

**★ 본 결함의 대상 = `:216-220`**:
```
// PLAN 0.23.11 ①: claude statusline — ` │ ` segment sep + ⚡ or 🧠.
// U+2502 (box light vertical), not ASCII |. Model name not hardcoded.
if (line.includes(" │ ") && (line.includes("⚡") || line.includes("🧠"))) {
  continue;
}
```
- 구분자는 **U+2502** ` │ `(양쪽 공백 필수), ASCII `|` 아님. 모델명 하드코딩 안 함(`Fable 5` 무관).
- 의도적 보수: ⚡/🧠 **없는** ` │ ` 줄은 보존(콘텐츠 오탐 방지 — 테스트 `:81-84`).

### `selectCardSummaryLine(chromeFiltered)` — `:253-272` (export)
chrome-필터된 텍스트에서 summary 1줄 선택. 말미→앞 순회하며 타이밍-전용 줄
(`WORKED_TIMING_LINE_RE`·`CLAUDE_TIMING_LINE_RE`) 스킵, 말미 TUI 타임스탬프 소거(`refineSummaryCandidate:241`).
유일 프로덕션 호출부 = `finishCard:1994` summary 분기(락 경계 근거 — §5).

## 6. 수정 스코프 제안 (옵션 — 결정은 아키텍트)

전제: 대상 상태줄 `Fable 5 ⚡high 🧠 │ …`는 **이미 `:218`이 커버**. 아래는 그 위에서의 선택지.

**옵션 A — 항목 종결 (무 코드변경) · 추천**
- 근거: 결함이 0.23.11 ①에서 해소·테스트 커버·재스모크 실증(impl-notes Implemented). 핸드오프 잔존
  표기(HANDOFF L81·L0-c)는 스테일. 신규 변종 실측이 없으면 열 게 없다.
- 조치: HANDOFF의 "claude 상태줄 chrome" 잔존 줄을 "0.23.11 ① `93c...` R35에서 해소" 완료로 정정
  (문서 정리 = §5.1 "불필요" 등급, 서브에이전트 위임 가능). R{n} 불요.

**옵션 B — 신규 변종이 실측되면 후속 후보 등재 (개별 보정 아님)**
- 잔존 갭(현행 `:218` 조건이 못 잡는 클래스, 이론상):
  - (b1) TUI 줄바꿈으로 ` │ ` 구분자와 ⚡/🧠가 **다른 시각 줄**에 렌더되어 어느 줄도 단독으론 조건 미충족.
  - (b2) ⚡/🧠 **둘 다 없는** 상태줄 변종(effort 표시 off 등) → `:81-84` 경계 테스트가 의도적으로 보존
    (콘텐츠 줄 `Foo │ bar` 오탐 방지) → 이 클래스는 필터 확장 시 **오탐 리스크 상승**이 대가.
- 처리: Deviations §0.23.11 누적 원칙에 따라 **개별 필터 추가가 아니라 후속 후보로 등재**하고 잔재 수용이
  기존 독트린. 개별 보정은 "보수 필터 망라 수집 아님" 원칙 위반.

**옵션 C — 지금 필터를 선제 강화** (비추천)
- 예: `⚡`/`🧠` 없이도 ` │ `+모델명 패턴이면 drop. → (b2) 오탐 표면 확대. R31 M-1 경계 자체는 안전
  (여전히 summary/inline만)하나 콘텐츠-포함 줄 과필터 리스크가 누적 원칙·보수 설계와 충돌. R{n} 권장될 만큼
  판단 무게 있음(오탐 트레이드오프).

**추천 1줄**: 옵션 A(항목 종결) — 결함은 0.23.11 ①(R35)에서 이미 해소, 핸드오프 잔존 표기가 스테일이므로
